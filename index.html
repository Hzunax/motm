Browser support?

As you can see, CSS grid is now supported in all major browsers (or at least partially supported). But this wasn't always the case. As many of you may know, it usually takes some time before browsers other than Chrome include the newest features. But does that mean you should wait to include those features in your projects?
Not necessarily. We are Exellysts, so we want to stay ahead of the crowd! Should we then exclude certain users because they are using an inferior browser? That's a definite no! Luckily, CSS provides a at-rule that you can use:

CSS @supports rule


								@supports (display: grid) {
								  div {
								    display: grid;
								  }
								}
							

								div {
								  display: flex;
								}
Using the @supports rule will check whether a browser supports that specific feature. If not, you can specify some fallback layout that will then be used in browsers that haven't included that feature.

But ...

What if @supports is not supported?
Well luckily, if the browser doesn't recognize the @supports rule, it will not even try to interpret it at all. It will simply discard the entire rule and move on to the next. Since you have defined some fallback content, it will take that as the styling.

Accessibility?

Web accessibility basicly means that people with disabilities should be able to interact with or access your website.
In Norway, inaccessible sites are illegal by law since July 2014. The standard that they have to follow is based on the Web Content Accessibility Guidelines 2.0 level AA.

Contrast ratio > 4.5

Relative luminance: 0 -> 1

Contrast ratio = (luminance of lighter color + 0.05) / (luminance of darker color + 0.05)

One of these guidelines says that the contrast ratio of text and images of text has to be at least 4.5:1 (3:1 for large text). How is this ratio calculated? Well, every color has a relative luminance. This value ranges from 0 for black, to 1 for white. Aggressive Web Apps

@philnash

The last talk we'll summarize for you is one by Phil Nash: Aggressive Web Apps.

Push notifications in the web

This one is about using push notifications on the web. Not how to code them, but rather how and when to use them.

Examples of good notifications

Notifications in your browser can be a good thing. You don't always want to have to download a full native app just to use part of their features. Examples: - Delivery service (pizza, package, ...) - Event updates (festival, conference, ...) so you don't always have to check their social media for updates. - Chat alerts (although some might find these annoying/unnecessary).

Timely, Actionable and Personal

According to Phil, good push notifications have to be Timely, Actionable and Personal.

Timely

Time of day

Time zone

Number of notifications

Consider the time of day (person might be asleep?), the time zone they're in and also the number of notifications you're sending that person as well.

Actionable

The notification shouldn't just be a message. If I receive a notification that I can check in for my flight now, I want to be able to click the notification and go to the check in procedure.

Personal

Bob sent you a message

You can check in to your flight now

Your pizza has arrived

It's important that the notification is actually about YOU.

Where do things go Wrong?

Many of you may recognize a pop up like this. It's the first time you're accessing this website (see url), and without context, they want to send you push notifications.

Phil's summary

At the end of his talk, Phil summarizes the most important things to keep in mind when using push notifications.

Don't ask notification permissions on page load

Allow users to opt out

You don't HAVE to make the notifications timely, actionable and personal. But if you don't, make sure the user can opt out from WITHIN your application, so they don't turn it off for all web apps!

If users block all notifications, the whole web suffers

Because if too many websites keep request permissions to send notifications without context, a user is just one click away of blocking ALL notifications.

Why conferences?

Now we gave a quick overview of the most interesting subjects during both conferences, we go a little bit more general. We want to show you that conferences are also a good way to improve your skills.

Thanks Exellys!

We get the awesome opportunity of Exellys to spend a couple of our workdays on events to improve our knowledge.
We are both front end developers at ING, working with the same Javascript library: Polymer. Polymer's aim is to make the life of developers easier when creating Progressive Web Apps with Web Components, service workers & HTTP/2. In our first months at our job, we did a 4 days training in this technology. This was very interesting and really went in to the details of the framework. But a couple of months later, we decided to do something else. This time, not a training, but a conference. Our goal was to explore different options for our open program and afterwards see which of both suits us the best, trainings or conferences.

Networking !important

We enjoyed the conference a lot. The atmosphere was good, everyone was very kind and open for conversation. Networking is a big part of conferences. Every kind of social media is used to enlarge networks. And that is what we also tried to do, network as much as possible. It is also nice that after the conference, you can still see updates of those people on their twitter accounts and learn even more new things.

Presentation1 !== Presentation2

The presentations on the conference itself can differ a lot. Some subjects can be very interesting for you and others maybe cover a thing that you do not really want to learn more about. Often, there are multiple presentations at the same moment, so normally, you can choose which one suits you the best. Besides that, the quality of the presentations also depends on the presentation skills of the person in front itself. Those persons have some experience in it, but still, not everyone is used to talk in front of a big audience, especially not when that audience exists out of sceptical developers.

training.value > conference.value

||

training.value < conference.value

So, while we're not claiming trainings should be skipped and conferences are the way to go, we do believe that combining both can add a lot of value to your profile. We have done both now, and we can see advantages and disadvantages in both of them. Trainings are very concrete, they go much more in to detail and will help you in mastering a quality in a short period of time. But on the other hand, in a rapidly changing world like the programming community, trainings sometimes fail to stay up to date. Most conferences won’t always provide that much information about one specific topic, but will help you in making the bigger picture and trigger you with the newest cool features.

T

Where trainings evolve your I-profile, conferences are better to improve your T-profile. As Exellysts, we should make that IT as big as possible.
